Base de datos

La base de datos está diseñada de forma relacional y normalizada.

  • users -> información común de todos los usuarios

  • roles / model_has_roles -> control de permisos

  • pacientes / especialistas -> datos específicos por rol

  • citas -> gestión unificada de consultas y citas

  • historiales -> información médica

  • documentos -> archivos asociados a historiales

  • logs -> auditoría de acciones
